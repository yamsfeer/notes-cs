回溯算法

n皇后问题

nnn皇后问题：在n∗nn*nn∗n的棋盘上放置nnn个皇后，皇后之间不可互相攻击（不在同行、同列或同斜线上），求有多少种布局。

nnn皇后问题的解可以用nnn维向量<x1,…,xn><x_1,\dots,x_n><x1​,…,xn​>来表示，每个元素表示的是棋盘的列。

经典的8皇后问题有92个解，其中一个是<1,5,8,6,3,7,2,4><1,5,8,6,3,7,2,4><1,5,8,6,3,7,2,4>。

以4皇后问题为例，它的解有<2,4,1,3><2,4,1,3><2,4,1,3>和<3,1,4,2><3,1,4,2><3,1,4,2>。

搜索空间

求解nnn皇后问题的过程可以用一个nnn叉树表示，4皇后问题的搜索空间是一个4叉树。

从根节点开始，每个节点有4个子节点，分别代表4列，虚线表示冲突（可以攻击），一旦有冲突，即可回溯。

叶子节点表示一个解，按深度优先搜索遍历树，可以得到所有解。

旅行问题

有nnn个城市，已知任意两个城市间距离，求一条恰好经过每个城市一次的回路，且长度最小。

举个例子，假设有ABCD四个城市，它们之间的距离关系如下图，它的解为<A,B,D,C><A,B,D,C><A,B,D,C>，总长度为23。

求解旅行问题的搜索空间是一个排列树，共有(n−1)!(n-1)!(n−1)!个叶子节点。

假设从A城市开始，此时有三种可能的选择，每选择一个城市，可选的城市跟着递减。

算法思想

总结下上面三个问题的特点：

  • 它们的解都是一个向量
  • 搜索空间都是树，可行解在叶子节点
  • 有深度优先、广度优先等遍历方法

n皇后可行解棋盘列号nnn叉树深度、广度优先彼此不攻击
0-1背包最优解是否放入背包子集树深度、广度优先不超背包重量

回溯算法适用于求解搜索问题和优化问题。

  • 搜索空间：树，节点对应部分解向量，可行解在树叶上

  • 搜索过程：深度、广度优先或其他方法遍历搜索树

  • 节点分支判定条件：当不满足约束条件，回溯到该节点的父节点

在搜索过程中，可以用黑白灰三色表示来节点状态：

  • 白节点：未访问
  • 灰节点：正在访问该节点为根的子树
  • 黑节点：该节点为根的子树遍历完成

适用条件

能使用回溯算法解决的问题，都满足多米诺性质。

P(x1,…,xk+1)→P(x1,…,xk)0<k<n P(x_1,\dots,x_{k+1}) \to P(x_1,\dots,x_k) \quad 0 <k<n P(x1​,…,xk+1​)→P(x1​,…,xk​)0<k<n

及其逆否命题：

¬P(x1,…,xk)→¬P(x1,…,xk+1)0<k<n \neg P(x_1,\dots,x_{k}) \to \neg P(x_1,\dots,x_{k+1}) \quad 0 <k<n ¬P(x1​,…,xk​)→¬P(x1​,…,xk+1​)0<k<n

简单来说，就是当检索到位置kkk时，如果不满足，继续往下肯定也是不满足，此时可以回溯；如果位置k+1k+1k+1满足，则它的上一步kkk一定是满足的。

以0-1背包问题为例，有10个物品：当放入1，2，3号物品时已经超重，则后面再继续放肯定也超重，此时需要回溯到2号物品；同理，如果放入1，2，3号时不超重，则只放入1，2号肯定不超重。

一个反例

求不等式的整数解：

5x1+4x2−x3≤10,1≤x1,x2,x3≤3 5x_1+4x_2-x_3 \le 10, \quad 1\le x_1,x_2,x_3 \le 3 5x1​+4x2​−x3​≤10,1≤x1​,x2​,x3​≤3

因为含有减法，这个问题不满足多米诺性质：

5x1+4x2−x3≤10⇏5x1+4x2≤10 5x_1+4x_2-x_3 \le 10 \nRightarrow 5x_1+4x_2\le 10 5x1​+4x2​−x3​≤10⇏5x1​+4x2​≤10

可以通过一个变换消除减法，使得其满足多米诺性质：

令 x3=3−x3′x_3=3-x_3'x3​=3−x3′​，原问题可以转换为：

5x1+4x2+x3′≤13,1≤x1,x3≤3,0≤x3′≤2 5x_1+4x_2+x_3' \le 13, \quad 1 \le x_1,x_3 \le 3, \quad 0 \le x_3' \le 2 5x1​+4x2​+x3′​≤13,1≤x1​,x3​≤3,0≤x3′​≤2

此时满足多米诺性质，可以用回溯法解决。

回溯算法：递归

ReBack(k) {
  if (k > n)
    return <x1,...,xn> // <x1,...,xn>是解
  while(S_k not empty) {
    x_k = S_k 中的最小值
    S_k = S_k - {x_k}
    ReBack(k+1)
  }
}

ReBacktrack(n) {
 	for(k from 1 to n) {
  	S_k = X_k
  }
  ReBack(1)
}
