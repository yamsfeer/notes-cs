分治算法

算法思想

分治算法的基本思想是：将大问题划分为若干个独立小问题，合并小问题的解可以得到大问题的解。

分治算法的3个步骤：

  1. 划分：将原始问题划分或归结为规模较小的子问题
  2. 独立求解子问题：递归或迭代求解每个子问题
  3. 综合得解：将子问题的解综合得到原问题的解

需要注意：

  1. 子问题与原始问题性质完全一样（分治基础）
  2. 子问题之间可彼此独立求解
  3. 递归停止时子问题可直接求解（重要特点）

分治算法的例子

二分查找

问题：在已排序的数组AAA中查找目标数xxx

设计思想：

  • xxx与中位数比较，将原问题归结为规模减半的子问题
  • 对子问题进行二分查找（注意点1）
  • 当子问题规模为1时，直接比较（注意点3）
// 算法 Binary Search(A, l, r, x)
// 输入：已排序数组A，下标从l到r，查找数x
// 输出：查找数下标或-1
while l <= r
  m = floor((l + r) / 2) // 中间划分
  if (A[m] == x)
    return m
  else if (A[m] > x)
    r = m - 1
  else
    l = m + 1
return -1

递推方程：W(n)=W(⌊n/2⌋)+1,W(1)=1W(n) = W(\lfloor n / 2 \rfloor) + 1, \quad W(1) = 1W(n)=W(⌊n/2⌋)+1,W(1)=1

时间复杂度：W(n)=⌊logn⌋+1W(n) = \lfloor logn \rfloor + 1W(n)=⌊logn⌋+1

二分归并排序

问题：长度为nnn的数组排序

设计思想：

  • 将原问题划分为规模为n / 2的2个子问题，继续划分，将原问题划分为规模为n / 4的4个子问题，继续划分直到规模为1
  • 规模为1时已排好序（注意点3）
  • 从规模1到规模n / 2，子问题排序后逐层向上归并，最终得到排序数组
// 算法 MergeSort(A, l, r)，对数组A中l到r的元素进行归并排序
// 输入：数组A[l...r]
// 输出：从小到大排序后的数组A
if (l < r)
  m = floor((l + r) / 2) // 中间划分
  MergeSort(A, l, m) // 子问题1
  MergeSort(A, m + 1, r) // 子问题2
  Merge(A, l, m, r) // 综合解

递推方程：W(n)=2W(n/2)+(n−1),W(1)=0W(n) = 2W(n / 2) + (n - 1), \quad W(1) = 0W(n)=2W(n/2)+(n−1),W(1)=0

时间复杂度：W(n)=nlogn−n+1W(n) = nlogn - n + 1W(n)=nlogn−n+1

汉诺塔问题

设计思想：

  • 将原问题归结为规模为n - 1的2个子问题，继续归约直到规模为1
  • 规模为1时，直接移动（注意点3）
  • 从规模1到n - 1，陆续组合两个子问题的解，直到规模为n
// 算法Hanoi(A, C, n), 将n个盘子从A移动到C
if n = 1
  move(A, C)
else
	Hanoi(A, B, n - 1) // 将n - 1个盘子从A移动到B
	move(A, C) // 将一个盘子从A移动到C
	Hanio(B, C, n - 1) // 将n - 1个盘子从B移动到C

递推方程：T(n)=2T(n−1)+1,T(1)=1T(n) = 2T(n - 1) + 1, \quad T(1) = 1T(n)=2T(n−1)+1,T(1)=1

时间复杂度：T(n)=2n−1T(n) = 2 ^ n - 1T(n)=2n−1

一般描述和分析方法

一般性描述

// Divide-and-Conquer(P)
if |P| <= c
  S(P)  // 当问题P的规模足够小时，直接求解
divide P into P1,P2,P3...Pk // 将问题P划分为若干个独立的子问题（划分）
for i from 1 to k // 对每个子问题递归求解（独立求解子问题）
  yi = Divide-and-Conquer(Pi)
return Merge(y1,y2,y3...yk) // 合并子问题的解得原始问题的解（综合解）

设计要点

  • 原问题可以划分为规模较小的子问题
    • 子问题与原问题具有相同的性质
    • 子问题的求解彼此独立
    • 划分时子问题的规模尽可能均衡
  • 子问题规模足够小时可直接求解
  • 子问题的解综合得到原问题的解

分治算法通常有递归或迭代两种算法实现。

分治的递推方程

上面提到，分治算法的大概是划分、求解子问题、综合解三步，它的递推方程为：

W(n)=W(P1)+⋯+W(Pk)+f(n),W(c)=C W(n) = W(P_1)+\dots+W(P_k)+f(n), \quad W(c) = C W(n)=W(P1​)+⋯+W(Pk​)+f(n),W(c)=C

其中W(P1)…W(Pk)W(P_1) \dots W(P_k)W(P1​)…W(Pk​)为求解各个子问题的工作量，f(n)f(n)f(n)为划分、综合解的工作量，W(c)W(c)W(c)是可直接求解子问题时的工作量。

分治算法中划分子问题有两种情况：

  • 子问题的个数按常数次减少

    汉诺塔问题的递推方程为：W(n)=2W(n−1)+1W(n) = 2W(n-1)+1W(n)=2W(n−1)+1

    这类算法的时间复杂度求解公式为：f(n)=∑i=1kaif(n−i)+g(n)f(n) = \sum ^k _{i=1} {a_if(n-i)} + g(n)f(n)=∑i=1k​ai​f(n−i)+g(n)

  • 子问题个数按倍数划分

    归并排序的递推方程为：W(n)=W(n/2)+1W(n) = W(n/2)+1W(n)=W(n/2)+1

    二分查找的递推方程为：W(n)=2W(n/2)+n−1W(n) = 2W(n/2)+n-1W(n)=2W(n/2)+n−1

    这类算法的时间复杂度求解公式为：f(n)=af(nb)+d(n)f(n) = af(\frac {n} {b}) + d(n)f(n)=af(bn​)+d(n)

快速排序

问题：对长度为nnn的数组AAA排序

基本思想：用首元素xxx作为划分标准，将AAA划分成两个子数组，小于等于xxx的组成ALA_LAL​，大于xxx的组成ARA_RAR​，递归地对ALA_LAL​和ARA_RAR​进行排序，直到子问题规模为1。

// QuickSort(A, l, r)
if (l < r)
	p = Partition(A, l, r) // 划分为两个子数组
  swap(A[l], A[p])
  QuickSort(A, l, p - 1)
  QuickSort(A, p + 1, r)

// Partition(A, l, r)
x = A[l] // 以首元素为分界点
i = l
j = r + 1
while(true)
  repeat j-- until A[j] <= x // 找到小于分界点的元素
  repeat i++ until A[i] > x // 找到大于分界点的元素

  i < j
		? swap(A[i], A[j]) // 交换
    : return j

小结

快速排序算法使用分治策略，它的子问题划分通常是由首元素决定的（也可以用其他方式决定）。

快速排序在最坏情况下的时间复杂度为O(n2)O(n^2)O(n2)，平均时间复杂度O(nlogn)O(nlogn)O(nlogn)。

nnn为偶数：an=an=an/2×an/2a^n = a ^ n = a ^ {n / 2} \times a ^ {n / 2}an=an=an/2×an/2

nnn为奇数：an=a(n−1)/2×a(n−1)/2×aa^n = a ^ {(n - 1) / 2} \times a ^ {(n - 1) / 2} \times aan=a(n−1)/2×a(n−1)/2×a

算法改进

对于分治的时间W(n)=aW(n/b)+f(n)W(n) = aW(n / b) + f(n)W(n)=aW(n/b)+f(n)，可通过减小aaa或减小f(n)f(n)f(n)来改进。其中：

  • aaa：子问题个数

  • n/bn/bn/b：子问题规模

  • f(n)f(n)f(n)：划分和综合解的工作量

我们可以通过减少子问题个数或通过预处理减少f(n)f(n)f(n)来改进分治算法。

减少子问题个数

减少子问题个数的方法：利用子问题的依赖关系，使某些子问题的解通过组合其他子问题的解而得到。

整数位乘问题

问题：X,YX,YX,Y是两个nnn位的二进制数，其中n=2kn = 2^kn=2k，求XYXYXY

  • 普通乘法（蛮力算法）：按位相乘，需要O(n2)O(n^2)O(n2)次位乘运算

  • 简单划分（未改进的分治）：

    令X=A2n/2+B,Y=C2n/2+DX = A2^{n/2} + B,\quad Y = C2^{n/2} + DX=A2n/2+B,Y=C2n/2+D，那么

    XY=AC2n+(AD+BC)2n/2+BD XY = AC2^n + (AD + BC)2^{n/2} + BD XY=AC2n+(AD+BC)2n/2+BD

    比如X=10110110,Y=10001110X = 10110110,Y=10001110X=10110110,Y=10001110

    此时原问题被划分为4个子问题AC,AD,BC,BDAC,AD,BC,BDAC,AD,BC,BD。

    递推方程和时间复杂度为：W(n)=4W(n/2)+O(n)→W(n)=O(n2)W(n) = 4W(n/2)+O(n) \to W(n) = O(n^2)W(n)=4W(n/2)+O(n)→W(n)=O(n2)

  • 改进：利用代数变换减少子问题个数

    因为AD+BC=(A−B)(D−C)+AC+BDAD + BC = (A-B)(D-C)+AC+BDAD+BC=(A−B)(D−C)+AC+BD，所以将上述等式右部代入原方程，有

    XY=AC2n+[(A−B)(D−C)+AC+BD]2n/2+BD XY = AC2^n + [(A-B)(D-C)+AC+BD]2^{n/2} + BD XY=AC2n+[(A−B)(D−C)+AC+BD]2n/2+BD

    此时子问题减少到三个：AC,(A−B)(D−C),BDAC,(A-B)(D-C),BDAC,(A−B)(D−C),BD。

    递推方程：W(n)=3W(n/2)+cn,W(1)=1W(n) = 3W(n/2)+cn, \quad W(1) = 1W(n)=3W(n/2)+cn,W(1)=1

    时间复杂度：W(n)=O(n1.59)W(n) = O(n^{1.59})W(n)=O(n1.59)
