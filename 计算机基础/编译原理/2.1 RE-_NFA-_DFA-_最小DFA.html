RE->NFA->DFA->最小DFA

要实现正则表达式，需要将RE逐步转换成最小DFA。它们之间的转换关系和相关算法如下图

我们先来回想下RE的定义和表示方法：

对给定的字符集Σ={c1,c2,…,cn}\Sigma = \{ c_1, c_2, \dots, c_n \}Σ={c1​,c2​,…,cn​}，归纳定义：

  • 空串 ε\varepsilonε 是正则表达式
  • 对于任意 c∈Σc \in \Sigmac∈Σ，ccc 是正则表达式
  • 如果 MMM 和 NNN 是正则表达式，则以下也是正则表达式
    • 选择 M ∣ N={M,N}M \:| \: N = \{M,N \}M∣N={M,N}
    • 连接 MN={mn ∣ m∈M,n∈N}MN = \{ mn \:|\: m \in M, n \in N \}MN={mn∣m∈M,n∈N}
    • 闭包 M∗={ε,M,MM,MMM,… }M^* = \{ \varepsilon, M,MM,MMM,\dots\}M∗={ε,M,MM,MMM,…}

RE的形式表示：

e ->
   | ε
   | c        (基本形式)
----------------------
   | e | e    (归纳形式)
   | e e
   | e*

总的来说，Thompson算法有一种类似数学归纳法的思想。

    • 对归纳（复合）形式的 RE 递归构造

子集构造算法

NFA的状态转移不确定，在处理字符时往往需要回溯。我们一般使用子集构造算法，将NFA转换成DFA， 得到确定的状态转移图。

构造与正则表达式A(B|C)*等价的NFA例子中，我们已经通过Thompson算法得到了与A(B|C)*等价的NFA。

下面将NFA转换为DFA（以下用sns_nsn​表示状态n。）。

ε\varepsilonε空串边表示一种零代价的转换，s1s_1s1​可以在没有任何输入的情况下转入s2s_2s2​，也就是说s1s_1s1​和s2s_2s2​是等价的，因此可以用一个新状态qqq来表示他们，q={s1,s2}q= \{s1, s2\}q={s1,s2}。

用这种方法，可以消除NFA中所有的ε\varepsilonε空串边，S={q0,…,qn}S = \{q_0, \dots,q_n\}S={q0​,…,qn​}就是DFA的状态集合。

  • 记q0={s0}q_0 = \{s_0\}q0​={s0​}

  • q0−A→q1,q1={s1,s2,s3,s4,s6,s9}q_0 -A\to q_1, \quad q_1 = \{s_1,s_2,s_3,s_4,s_6,s_9\}q0​−A→q1​,q1​={s1​,s2​,s3​,s4​,s6​,s9​}

  • q1−B→q2,q2={s5,s8,s9,s3,s4,s6}q_1 -B\to q_2, \quad q_2=\{s_5,s_8,s_9,s_3,s_4,s_6\}q1​−B→q2​,q2​={s5​,s8​,s9​,s3​,s4​,s6​}

  • q1−C→q3,q3={s7,s8,s9,s3,s4,s6}q_1 -C\to q_3, \quad q_3=\{s_7,s_8,s_9,s_3,s_4,s_6\}q1​−C→q3​,q3​={s7​,s8​,s9​,s3​,s4​,s6​}

  • q2−B→{s5,s8,s9,s3,s4,s6}=q2q_2 -B\to \{s_5,s_8,s_9,s_3,s_4,s_6\} = q_2q2​−B→{s5​,s8​,s9​,s3​,s4​,s6​}=q2​

  • q2−C→{s7,s8,s9,s3,s4,s6}=q3q_2 -C\to \{s_7,s_8,s_9,s_3,s_4,s_6\} = q_3q2​−C→{s7​,s8​,s9​,s3​,s4​,s6​}=q3​

  • 重复步骤，直到不能得出新的子集

  • S={q0,…,qn}S = \{q_0, \dots,q_n\}S={q0​,…,qn​}中，包含接受状态s9s_9s9​的都是DFA的接受状态

介绍两个重要的概念：边界和**ε\varepsilonε-闭包**：

  • 边界

    一个状态经过ε\varepsilonε能到达的所有状态的集合。比如s1s_1s1​的边界是{s2,s3,s4,s6,s9}\{s_2,s_3,s_4,s_6,s_9\}{s2​,s3​,s4​,s6​,s9​}

  • ε\varepsilonε-闭包

    将q1q_1q1​中每个元素经过一个非ε\varepsilonε字符后能到达的状态集合记为delta(q1)delta(q_1)delta(q1​)，那么delta(q1)={s5}delta(q_1) = \{s_5\}delta(q1​)={s5​}，

    对delta(q1)delta(q_1)delta(q1​)中每个元素求它的边界，得到的集合称为q1q_1q1​的**ε\varepsilonε-闭包**，记为eps−closure(q1)eps-closure(q_1)eps−closure(q1​)。

    不难得出，eps−closure(q1)=q2eps-closure(q_1) = q_2eps−closure(q1​)=q2​。

对算法的讨论

  • 不动点算法

    • 算法可以提前运行终止，因为状态数是有限的

  • 时间复杂度

    • 最坏的情况O(2N)O(2^N)O(2N)
    • 但是实际中不常发生，因为并不是每个子集都会出现
    • 上面的例子中，有s0s_0s0​到s9s_9s9​共十个元素，本来应该最多发生2102^{10}210次，实际上只发生了3次（s0,s1,s2s_0,s_1,s_2s0​,s1​,s2​）

ε-闭包的计算：深度优先

/* ε-closure 基于深度优先遍历的算法 */
set closure = {};
void eps_closure(x){
  closure += {x};  // 集合的加法， 并
  foreach(y: x --ε--> y){  // y 是 x 通过 ε 转换到的 y。
    if(!visited(y)){     // 如果 y 还没有访问过，就访问 y
      eps_closure(y);
    }
  }
}

ε-闭包的计算：宽度优先

/* ε-closure： 基于宽度优先遍历的算法 */
set closure = {};
Q = []; // quenu 基于队列的概念，
void eps_closure(x) =
  Q = [x];
  while(Q not empty)
    q <- deQueue(Q)
    closure += q
    foreach(y: q --ε--> y)  // 将所有的从 Q 开始可以走到的 y 都加到 Q 里
      if(!visited(y))
        enQueue(Q,y)

子集构造算法：工作表算法

q0 <- eps_closure(n0)   // q0 = {n0}
Q <- {q0}       // Q = {q0}
workList <- q0     // workList = [q0, ...]
while(workList != [])
  remove q from workList   // workList = [...]
  foreach(character c)     // c = a
    t <- e-closure(delta(q,c))   // delta(q0, a) = {n1}, t = {n1, n2, n3, n4, n6, n9}
    D[q,c] <- t    //   q1 = t
    if(t not in Q)    // Q = {q0, q1} , workList = [q1]
      add t to Q and workList

Hopcroft算法

我们已经通过子集构造算法得到了与A(B|C)*等价的DFA。

事实上，这个DFA还存在改进的空间。比如状态q2和状态q3可以合并为一个状态，合并之后还可以继续合并。

合并过程见下图：

最终，我们得到了最终的A(B|C)*等价的最小DFA 。

Hopcroft算法具体过程

算法过程

  1. 根据是否为接受状态将所有节点分为N和A两个集合。 N 为非接受状态，A 为接受状态

  2. 对N和A进行split划分，直到不能得出新的集合

  3. split划分

    split划分是基于等价类思想的一个过程。

    对于状态集合SSS和字符集合Σ\SigmaΣ，如果SSS中的状态s1,s2s_1,s_2s1​,s2​对Σ\SigmaΣ中的字符ccc的行为是一致的，则s1,s2s_1,s_2s1​,s2​对ccc是等价的，它们可以划分为一组。

    上图中，1,2,3经过A后分别进入S2和S3，那么S1可划分为{1,2}和{3}。

伪代码表示如下：

split(S)
  foreach(character c) // 对字符集中每个字符都做判断
    if(c can split s) // 字符c可以划分s
      split s into T1, ..., Tk // 划分为k组

hopcroft()
  split all nodes into N, A
  while(set is still changes) // 直到不出现新的集合
    split(s)

A(B|C)*例子

回忆下A(B|C)*例子在经过子集构造算法后得到的DFA：

  1. 根据是否接受划分为N={q0}, A={q1,q2,q3}N = \{q_0\}, \: A = \{q_1,q_2,q_3\}N={q0​},A={q1​,q2​,q3​}

  2. NNN不可再划分

  3. 观察AAA，可得以下几点

    • q1,q2,q3−B→q2,q2∈Aq_1,q_2,q_3 -B\to q_2, q_2 \in Aq1​,q2​,q3​−B→q2​,q2​∈A，所以B不可划分AAA
    • 同理，q1,q2,q3−C→q3,q3∈Aq_1,q_2,q_3 -C\to q_3, q_3 \in Aq1​,q2​,q3​−C→q3​,q3​∈A，所以C不可划分AAA

  4. 那么，q1,q2,q3q_1,q_2,q_3q1​,q2​,q3​对字符B和字符C都是等价的，可以用一个新状态q4q_4q4​等价替换它们。

  5. 等价替换和得到A(B|C)*的最小DFA

DFA的数据结构表示

从概念上讲，DFA是一个有向图，可以用邻接矩阵或邻接表表示。

实际上，有不同的DFA的代码表示，具体选择哪种取决于在实际情况对时间空间的权衡。

  • 转移表（类似于邻接矩阵）
  • 跳转表（大部分情况下，使用跳转表）
  • 哈希表
  • ...

// 这个例子中只有{A,B,C}三个字符。M = 2, N = 3
// 实际例子中，比如ASCII码，N = 256
char table[M][N];

table[0]['a'] = 1;
table[1]['b'] = 1;
table[1]['c'] = 1;
// ...

nextToken函数可以最长匹配字符流中下一个token：

nextToken()
  state = 0
  stack = [] // 用于实现最长匹配
  while(state != ERROR) //在上面的表格中表示为空
    c = getChar()
      if(state is ACCEPT)  // 接受状态,q1； 表格中表示为状态1，有（1,b）, (1,c)
        clear(stack)
    push(state)
    state = table[state][c]
  while(state is not ACCEPT)
    state = pop()
    rollback()

跳转表

同一个例子，用跳转表实现，伪代码如下：

nextToken()
  state = 0
  stack = []
  goto q0
q0:
  c = getChar()
  if(state is ACCEPT)
    clear(stack)
  push(state)
  if(c == 'a')
    goto q1
q1:
  c = getChar()
  if(state id ACCEPT)
    clear(stack)
  push(state)
  if(c == 'b' || c == 'c')
    goto q1

跳转表有两个优点：

  • 指定了转换的字符，可以提高转移的效率
  • 不需要维护转移表，节省内存。转移表在字符集大（比如Unicode）和状态多时，占用内存大

大部分情况下，我们会使用跳转表。
