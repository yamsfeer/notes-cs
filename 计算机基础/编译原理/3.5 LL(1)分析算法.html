                                                                  # LL(1)分析算法

                                                                  LL(1)分析算法的意思是：从左(Left)向右读入程序，选择最左(L)推导，采用一个(1)前看符号。

                                                                  跟递归下降分析算法相同，LL(1)分析算法也有分析高效（线性时间）、错误定位和诊断信息准确等优点。

                                                                  该算法可以应用于语法分析器生成器中，比如ANTLRopen in new window，类似的生成器还有YACC、bison等。

                                                                  Tips

                                                                  ANTLR是基于LL算法实现的语法解析器生成器，用Java语言编写。

                                                                  # 概述

                                                                  回顾之前自顶向下分析的例子，给定文法GGG如下，推导句子g d w：

                                                                  S -> N V N
N -> s
   | t
   | g
   | w
V -> e
   | d

                                                                  自顶向下的分析算法如下：

                                                                  tokens[];
i = 0;
stack = [S];
while(stack not empty)
  if(stack[top] is a terminal t)
    t == tokens[i++]
    	? pop()
    	: backtrack(); // 终结符t与目标字符不等，回溯
  else if(stack[top] is a nonterminal T)
    pop();
    push(the next right hand side of T); // 选择T的右边压栈

                                                                  backtrack函数是这个算法的回溯操作，这使得算法性能较低。而算法需要回溯，原因出在最后一行的压栈操作上。选择T的右边压栈是不能保证下一次匹配成功的。

                                                                  比如当推导gdw中的g时，会逐次压入N的右部，按照s，t，g，w 的顺序压入。其实可以直接压入g，如果g不匹配，则直接报错。

                                                                  如果每次都能把“正确的”那个压栈，就可以避免回溯。

                                                                  假设我们针对文法GGG有以下这张LL(1) 分析表：

                                                                  S -> N V N
N -> s
   | t
   | g
   | w
V -> e
   | d
                                                                  N \ Tstgwed
                                                                  S0000
                                                                  N1234
                                                                  V56

                                                                  行是非终结符，列是终结符（前看符号），表中数字对应的是文法中产生式规则标号（行号，从0开始）。

                                                                  每当推导遇到非终结符N时，可以根据非终结符N和当前前看符号查LL(1)分析表，得出正确的符号压栈。

                                                                  有了分析表，对句子g d w的分析过程如下：

                                                                  1. 栈顶元素为S
                                                                  2. S出栈，前看符号是g。查分析表，第0条产生式（S -> N V N）的右部，也就是N，V，N依次入栈。当前栈底到栈顶元素为N V N
                                                                  3. 栈顶N出栈，前看符号是g。查分析表，第3条产生式（N -> g）的右部入栈。当前栈底到栈顶元素为N V g
                                                                  4. 栈顶元素g是非终结符且g == 前看符号。g出栈，前看符号变为d
                                                                  5. 重复上述步骤

                                                                  # FIRST集

                                                                  还是这个主谓宾的例子：

                                                                  S -> N V N
N -> s
   | t
   | g
   | w
V -> e
   | d

                                                                  在这个文法中，句子都以NNN开头。而NNN能推导出{s,t,g,w}\{s, t, g, w\}{s,t,g,w}，那么所有句子必然以{s,t,g,w}\{s, t, g, w\}{s,t,g,w}集合中的其中一个字符开头。

                                                                  将这个集合称为FIRST(N)FIRST(N)FIRST(N)，有FIRST(N)={s,t,g,w}FIRST(N) = \{s, t, g, w\}FIRST(N)={s,t,g,w}。

                                                                  • 对产生式N→abcN \to abcN→abc，有$FIRST(N) = FIRST(N) \cup {a} $
                                                                  • 对产生式N→MbcN \to MbcN→Mbc，有$FIRST(N) = FIRST(N) \cup FIRST(M) $

                                                                  # FIRST集的近似不动点算法

                                                                  求一个文法中所有非终结符的FIRST集的伪代码如下：

                                                                  foreach (nonterminal N) // 对文法中所有的非终结符做循环
  FIRST(N) = {}

while (some set is changing) // 循环至不再变化
  foreach (production p: N -> β1...βn) // 对所有产生式做循环
    if (β1 is terminal t) // 如果产生式以终结符t开头
      FIRST(N) U= {t}
    else if (β1 is nonterminal M) // 如果产生式以非终结符M开头
      FIRST(N) U= FIRST(M)

                                                                  针对主谓宾的例子，计算过程用表表示如下：

                                                                  N\FIRST0（初始）1次循环2次循环3次循环
                                                                  S∅\emptyset∅∅\emptyset∅{s,t,g,w}\{s,t,g,w\}{s,t,g,w}不再变化
                                                                  N∅\emptyset∅{s,t,g,w}\{s,t,g,w\}{s,t,g,w}不再变化
                                                                  V∅\emptyset∅{e,d}\{e,d\}{e,d}不再变化

                                                                  由此可得表：

                                                                  NFIRST(N)
                                                                  S{s,t,g,w}\{s, t, g, w\}{s,t,g,w}
                                                                  N{s,t,g,w}\{s, t, g, w\}{s,t,g,w}
                                                                  V{e,d}\{e, d\}{e,d}

                                                                  # 把 FIRST 集推广到任意串上

                                                                  上面计算的是对非终结符的FIRSTFIRSTFIRST集的计算方法。

                                                                  把FIRST集推广到任意产生式，有：

                                                                  • FIRST_S(β1…βn)=FIRST(N),β1=NFIRST\_S(\beta_1\dots\beta_n) = FIRST(N), \quad \beta_1 = NFIRST_S(β1​…βn​)=FIRST(N),β1​=N​​

                                                                  • FIRST_S(β1…βn)=t,β1=tFIRST\_S(\beta_1\dots\beta_n) = {t}, \quad \beta_1 = tFIRST_S(β1​…βn​)=t,β1​=t​​​​

                                                                  有了上面非终结符的FIRSTFIRSTFIRST集表，不难得出各产生式的FIRST_SFIRST\_SFIRST_S集：

                                                                  S -> N V N  // { s, t, g, w }
N -> s      // { s }
   | t      // { t }
   | g      // { g }
   | w      // { w }
V -> e      // { e }
   | d      // { d }

                                                                  上面我们已经得到了整个文法GGG的FIRST_SFIRST\_SFIRST_S集：

                                                                  0: S -> N V N  // { s, t, g, w }
1: N -> s      // { s }
2:    | t      // { t }
3:    | g      // { g }
4:    | w      // { w }
5: V -> e      // { e }
6:    | d      // { d }

                                                                  以产生式S -> NVN为例，如果当前前看符号是s、t、g、w的话，则将S -> NVN的行号0入栈。

                                                                  由此我们可以整理得到LL(1)分析表如下：

                                                                  N \ Tstgwed
                                                                  S0000
                                                                  N1234
                                                                  V56

                                                                  # LL(1) 分析表中的冲突

                                                                  对与一个文法GGG构建出的LL(1)分析表，如果

                                                                  • 每个表项最多只有一个元素，GGG称为LL(1)文法。
                                                                  • 每个表项的元素个数大于1，称为LL(1)分析表的冲突。

                                                                  假设有如下文法GGG：

                                                                  0: S -> N V N  // { s, t, g, w }
1: N -> s      // { s }
2:    | t      // { t }
3:    | g      // { g }
4:    | w      // { w }
5:    | w V    // { w } 注意这行
6: V -> e      // { e }
7:    | d      // { d }

                                                                  相对应的分析表如下：

                                                                  N \ Tstgwed
                                                                  S0000
                                                                  N1234, 5
                                                                  V67

                                                                  当非终结符为N且前看符号为w时，有4和5两种选择。

                                                                  这样的冲突会导致分析过程的不确定性，导致回溯的发生。

                                                                  对NNN的两条产生式 N→βN \to \betaN→β​ 和 N→γN \to \gammaN→γ​​​，如果

                                                                  FIRST_S(N→β)∩FIRST_S(N→γ)=∅ FIRST\_S(N \to \beta) \cap FIRST\_S(N \to \gamma) = \emptyset FIRST_S(N→β)∩FIRST_S(N→γ)=∅

                                                                  则没有冲突。

                                                                  然而：

                                                                  FIRST_S(N→w)∩FIRST_S(N→wV)={w}≠∅ FIRST\_S(N \to w) \cap FIRST\_S(N \to w V) = \{w\} \ne \emptyset FIRST_S(N→w)∩FIRST_S(N→wV)={w}=∅

                                                                  文法GGG不是一个LL(1)文法。

                                                                  # 一般情况下的LL(1)分析表

                                                                  假设有如下文法（下文中所有例子都使用这个文法）：

                                                                  Z -> d
   | X Y Z
Y -> c
   | ε
X -> Y
   | a

                                                                  在求Z→XYZZ \to XYZZ→XYZ​的FIRST_SFIRST\_SFIRST_S​集时，由于YYY​可以推导出ε\varepsilonε​空串，而XXX​可以推导出YYY​，最终FIRST_S(Z→XYZ)={a,c,d}FIRST\_S(Z \to XYZ) = \{a, c, d\}FIRST_S(Z→XYZ)={a,c,d}​。

                                                                  由此可见产生式在能推导出空串时，情况会比较复杂。

                                                                  为解决这个问题我们需要两个新集合NULLABLENULLABLENULLABLE、FOLLOWFOLLOWFOLLOW：

                                                                  • NULLABLENULLABLENULLABLE​：可以推导出空串的非终结符集合
                                                                  • FOLLOW(N)FOLLOW(N)FOLLOW(N)​：非终结符NNN​​后面跟着的符号的集合

                                                                  # NULLABLE 集合

                                                                  非终结符NNN属于NULLABLENULLABLENULLABLE集，当且仅当

                                                                  • 基本情况：N→εN \to \varepsilonN→ε

                                                                  • 归纳情况：N→N1N2…Nn,N1N2…Nn∈NULLABLEN \to N_1N_2 \dots N_n , \quad N_1N_2 \dots N_n \in NULLABLEN→N1​N2​…Nn​,N1​N2​…Nn​∈NULLABLE​​​​​​

                                                                    N1…NnN_1 \dots N_nN1​…Nn​都是非终结符且属于NULLABLENULLABLENULLABLE集。

                                                                  # NULLABLE 集合算法

                                                                  NULLABLE = {};
while(nullable is still changing) {
  for(production p: N -> β) {
    if (β == epsilon) // 基本情况
      NULLABLE.add(N)
    if (β == N1 ... Nn) // 归纳情况
      if (N1 in NULLABLE && ... && Nn in NULLABLE)
        NULLABLE.add(N)
  }
}

                                                                  # FIRST 集合的完整计算公式

                                                                  有了上述NULLABLENULLABLENULLABLE的计算方式，我们可以推广FIRSTFIRSTFIRST​集的完整计算公式。

                                                                  • 基本情况：对产生式N→tN \to tN→t​​

                                                                    $FIRST(N) = FIRST(N) \cup {t} $​​

                                                                  • 归纳情况：对产生式N→N1,N2,…,NnN \to N_1,N_2,\dots,N_nN→N1​,N2​,…,Nn​

                                                                    • FIRST(N)=FIRST(N)∪FIRST(N1)FIRST(N) = FIRST(N) \cup FIRST(N_1)FIRST(N)=FIRST(N)∪FIRST(N1​)​
                                                                    • ifN1∈NULLABLE,FIRST(N)=FIRST(N)∪FIRST(N2)if \quad N_1 \in NULLABLE, \quad FIRST(N) = FIRST(N) \cup FIRST(N_2)ifN1​∈NULLABLE,FIRST(N)=FIRST(N)∪FIRST(N2​)​​​​​​
                                                                    • ifN1,N2∈NULLABLE,FIRST(N)=FIRST(N)∪FIRST(N3)if \quad N_1,N_2 \in NULLABLE, \quad FIRST(N) = FIRST(N) \cup FIRST(N_3)ifN1​,N2​∈NULLABLE,FIRST(N)=FIRST(N)∪FIRST(N3​)​​​​
                                                                    • …\dots…

                                                                  # FIRST 集的完整不动点算法

                                                                  for(nonterminal N)
  FIRST(N) = {}; // 对所有非终结符初始化FIRST集为空集
while(some set is changing) {
  for(production p: N -> β1 ... βn) {
    // 注意这里β1 ... βn指的是某条产生式的右部，比如N -> aBc，β1 == a
    for(βi from β1 to βn) { // 对aBc
      if (βi is terminal t) // βi是终结符
        FIRST(N).add(t);
        break;
      if (βi is nonterminal M) // βi是非终结符
        FIRST(N).add(FIRST(M))
        if (M is not in NULLABLE)
          break;
        else { // M属于NULLABLE集
          // ...
        }
    }
  }
}

                                                                  # FOLLOW 集的不动点算法

                                                                  foreach (nonterminal N)
  FOLLOW(N) = {}
while (some set is changing)
  foreach (production p: N -> β1 ... βn)
    temp = FOLLOW(N)     // temp 记录在 βn 的后面
    foreach (βi from βn downto β1)   // 逆序的 FOLLOW
      if(βi is terminal t)  // terminal
        temp = {t}
      if (βi is nonterminal M)    // nonterminal
        FOLLOW(M) U= temp
        if(M is not NULLABLE)
          temp = FIRST(M)
        else
          /* 如果M是NULLABLE，那么我们不仅仅能看到FIRST(M),
          	 还能M的FIRST(M)后面的元素，也就是非最右项的其余右项。*/
          temp U= FIRST(M)

                                                                  foreach (production p)
  FIRST_S(p) = {}
calculte_FIRST_S (production p: N -> β1 ... βn) {
  foreach (βi from β1 to βn) {
    if(βi is terminal t) // terminal
      FIRST_S(p) U= {t};
      return;
    if (βi is nonterminal M) // nonterminal
      FIRST_S(p) U= FIRST(M);
      if (M is not NULLABLE)
        return;
  }
  FIRST_S(p) U= FOLLOW(N)
}

                                                                  # LL(1) 分析冲突处理

                                                                  当分析表出现冲突时，该文法也许不适合使用LL(1)分析算法来进行语法分析。

                                                                  但在某些情况下，这些冲突是可以被消除的。下面介绍两种方法。

                                                                  # 消除左递归

                                                                  以下面这个文法为例：

                                                                  0: E -> E + T
1:    | T
2: T -> T * F
3:    | F
4: F -> n

                                                                  它的FIRST_SFIRST\_SFIRST_S表如下：

                                                                  01234
                                                                  FIRST_SFIRST\_SFIRST_S{n}\{n\}{n}​{n}\{n\}{n}{n}\{n\}{n}{n}\{n\}{n}{n}\{n\}{n}

                                                                  LL(1)分析表：

                                                                  n+ / *
                                                                  EEE0, 1（冲突）
                                                                  TTT2, 3（冲突）
                                                                  FFF4

                                                                  这个例子中，之所以会产生冲突，原因在产生式中存在左递归。比如第0条产生式E→E+TE \to E + TE→E+T，和第1条产生式E→TE \to TE→T。

                                                                  为了消除冲突，可以通过改写文法来消除左递归。

                                                                  将上述文法改写成：

                                                                  0: E -> T E'
1: E'-> + T E'
2:    | ε
3: T -> F T'
4: T'-> * F T'
5:    | ε
6: F -> n

                                                                  # 提取左公因子

                                                                  针对以下例子：

                                                                  0: X -> a Y
1:    | a Z
2: Y -> b
3: Z -> c

                                                                  提取公因子，改写成：

                                                                  0: X -> a X'
1: X'-> Y
2:    | Z
3: Y -> b
4: Z -> c

                                                                  # 总结

                                                                  从头整理一下整个过程：

                                                                  • 主要原因：选择T的右边压栈不能保证下一次匹配成功，这会导致需要回溯，使得程序性能低
                                                                  • 目的：消除回溯
                                                                  • 解决方法：用LL(1)分析表提示入栈对象
                                                                  • 为了构建LL(1)分析表
                                                                    • 需要针对每个非终结符的FIRSTFIRSTFIRST集
                                                                    • 子问题：一般情况下，产生式在能推导出空串时，情况会比较复杂
                                                                      • 解决子问题：使用NULLABLENULLABLENULLABLE和FOLLOWFOLLOWFOLLOW集
                                                                    • 有了FIRSTFIRSTFIRST​、NULLABLENULLABLENULLABLE​和FOLLOWFOLLOWFOLLOW​集，就可以构建针对每条产生式的FIRST_SFIRST\_SFIRST_S​集
                                                                  • 根据FIRST_SFIRST\_SFIRST_S集构造LL(1)分析表

                                                                  有了LL(1)分析表，最终的LL(1)分析器代码如下：

                                                                  tokens[];
i = 0;
stack = [S];
while(stack not empty)
  if(stack[top] is a terminal t)
    t == tokens[i++]
      ? pop()
      : error(); // 终结符t与目标字符不等，直接报错，不需要回溯
  else if(stack[top] is a nonterminal T)
    pop();
    // push(the next right hand side of T); // 选择T的右边压栈
    push(table[T, tokens[i]]) // 使用分析表来提示将什么压栈

                                                                  其分析过程，就是

                                                                  • 输入一个文法，得到相应的 LL(1) 分析表
                                                                  • 语法分析在 LL(1) 分析表的驱动下对输入的程序进行分析，判断程序是否符合文法的描述。

                                                                  至此，就是整个表驱动的LL分析器架构。

                                                                  不难看出，分析表是整个过程的核心。

                                                                  语法分析器生成器的作用就是根据语法规则生成分析表。
